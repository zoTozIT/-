# Система учёта заказов «Быстрая доставка»

Программа для учёта клиентов и заказов компании доставки.

Работает в двух режимах:
- **GUI** — обычное окно с кнопками (`main_gui.py`). Это основной режим, начни с него.
- **CLI** — управление текстовыми командами (`main_cli.py`): отчёты, экспорт, импорт.

Данные хранятся в базе данных — поддерживаются **два движка на выбор: SQLite (по
умолчанию) и TinyDB**. Есть отчёты, экспорт/импорт в JSON и XML, логирование и тесты.

## Запуск в PyCharm — пошагово

### Шаг 1. Открыть проект
**File → Open** и выбери папку **`delivery_system`** (именно её).

### Шаг 2. Добавить примеры данных (чтобы программа была не пустой)
1. Слева в списке файлов найди **`создать_примеры.py`**.
2. Правый клик по нему → **Run 'создать_примеры'** (зелёный треугольник ▶).
3. Внизу в консоли должно появиться:
   ```
   Демонстрационные данные добавлены.
   Готово! Теперь запусти main_gui.py — увидишь примеры заказов.
   ```

> ⚠️ **Внимание:** `создать_примеры.py` полностью **очищает базу** (удаляет всех
> клиентов и заказы) и заново добавляет 3 клиентов и 4 заказа, при этом нумерация
> id сбрасывается и начинается с 1. Запускай его только когда хочешь начать с чистого
> листа. Если у тебя уже есть нужные данные — этот шаг **пропусти** и добавляй всё
> вручную через кнопки в окне программы.

### Шаг 3. Запустить программу с окнами
1. Найди файл **`main_gui.py`**.
2. Правый клик → **Run 'main_gui'** ▶.
3. Откроется окно — в таблице будут примеры заказов.

> ⚠️ Если окно программы уже было открыто раньше (пустым) — **закрой его крестиком и запусти заново**. Окно показывает заказы на момент открытия и само не обновляется.

## Как пользоваться окном программы (GUI)

**Главное окно** — таблица заказов и кнопки.

- **Клиенты** — открыть список клиентов. Здесь можно добавить нового (имя, телефон, адрес) или удалить. Клиента нельзя удалить, если у него есть заказы.
- **Добавить** — создать новый заказ:
  1. Выбрать клиента из списка.
  2. Указать дату (по умолчанию сегодня).
  3. Выбрать статус (`новый`, `в доставке`, `выполнен`, `отменён`).
  4. Внизу вписать товар — название, количество, цену — и нажать **«+ позиция»** (товаров можно добавить несколько).
  5. Нажать **«Сохранить»**.
- **Редактировать** — выдели заказ в таблице и измени его.
- **Удалить** — удалить выделенный заказ.
- **Фильтр по статусу** — вверху: выбери статус и нажми **«Применить»**.
- **Показать отчёт** — статистика в отдельном окне (заказы по статусам, топ-3 клиента, выручка за период).
- **Экспорт / Импорт** — сохранить или загрузить данные. Рядом выпадающий список формата: **JSON** или **XML**.

> Если при создании заказа пишет, что нет клиентов — сначала добавь клиента через кнопку **«Клиенты»**.

## Режим команд (CLI) — для отчётов и экспорта

Нужен, если хочешь показать работу через консоль. Запускается из вкладки **Terminal**
внизу PyCharm (открой её и вписывай команды):

```
python main_cli.py report --period month      отчёт за месяц (day / week / month)
python main_cli.py list-orders                 список всех заказов
python main_cli.py list-orders --status новый  список с фильтром по статусу
python main_cli.py list-customers              список клиентов

python main_cli.py export --file backup.json   сохранить всё в файл JSON
python main_cli.py export --file backup.xml    сохранить всё в файл XML
python main_cli.py import --file backup.json   загрузить заказы из файла
```

Файлы экспорта появляются в папке проекта.

> Если `python` не срабатывает — попробуй писать `py` вместо `python`.

## Выбор базы данных: SQLite или TinyDB

По умолчанию используется **SQLite** (файл `data/delivery.db`) — ставить ничего не нужно.

Дополнительно поддерживается **TinyDB** (файл `data/tinydb.json`). Чтобы им
воспользоваться, сначала установи библиотеку — в Terminal:
```
pip install tinydb
```

**В режиме команд (CLI)** добавь флаг `--backend tinydb`:
```
python main_cli.py --backend tinydb seed
python main_cli.py --backend tinydb report --period month
```

**В режиме окон (GUI)** движок выбирается переменной окружения `DELIVERY_BACKEND`.
Проще всего в PyCharm: **Run → Edit Configurations…** → у `main_gui` в поле
**Environment variables** добавь `DELIVERY_BACKEND=tinydb`. Без неё работает SQLite.

> Оба движка дают одинаковый результат — это просто два разных способа хранения
> данных «под капотом». SQLite хранит данные в бинарном файле, TinyDB — в обычном
> JSON-файле, который можно открыть и прочитать глазами.

## Тесты (по желанию)

Сначала установи библиотеки — в Terminal:
```
pip install -r requirements.txt
```
Потом запусти тесты:
```
pytest                      запустить все тесты
pytest --cov=.              показать процент покрытия кода
```
Все тесты должны пройти (`passed`). Покрытие кода около **91%**. Тесты базы данных
автоматически прогоняются на обоих движках (SQLite и TinyDB), а GUI проверяется
отдельным набором тестов.

## Частые вопросы

**В окне пусто.** Запусти `создать_примеры.py`, затем **закрой и заново открой** `main_gui.py`.

**Как очистить базу?** Удали файл `data/delivery.db` — он создастся заново пустым при следующем запуске.

**Где логи?** В `logs/delivery.log` — туда пишутся все действия и ошибки.

**Кракозябры в консоли.** Это особенность кодировки консоли, на работу не влияет.
При желании выполни один раз в Terminal: `chcp 65001`.
